# Tech Stack Showcase

A personal tech stack website for showcasing development tools and applications. Built with Handlebars, TinaCMS, and deployed on Netlify.

## Features

- Clean, modern dark/light theme with auto system detection
- Live search and category filtering
- TinaCMS integration for visual content editing
- Fully responsive design
- Fast static output, no runtime server needed
- Keyboard shortcuts (Alt+0-9 for categories, Alt+A for all)

## Quick Start

### Local Development

```bash
npm install
node build.js && node dev-server.js
```

Visit `http://localhost:3000` to see the site with live reload.

> **Note**: `npm run dev` (which runs `tinacms dev`) may fail if the project is in a Dropbox path with special characters. Use the direct commands above instead.

### Production Build (Netlify)

Netlify runs automatically on push to `main`:
```
BUILD_DIR=dist node build.js && tinacms build
```

## Project Structure

```
content/tools/<category>/  Tool entries organized by category subfolder
content/categories.json   8 categories with icons and subcategories
content/settings.json     Site title, author, URLs
templates/index.hbs       Main page layout (Handlebars)
templates/partials/       Reusable components (tool-card.hbs)
src/styles/               main.css + themes.css
src/scripts/              theme.js, search.js, filter.js, modal.js
tina/                     TinaCMS config (Tina Cloud)
tina/__generated__/       Auto-generated by tinacms (DO NOT edit)
build.js                  Custom static site generator
dev-server.js             Express + Chokidar dev server (port 3000)
```

## Content Management

### Using TinaCMS (Recommended)

1. Go to [app.tina.io](https://app.tina.io) and sign in
2. Select the tech-stack project
3. Add or edit tools through the visual interface
4. Changes are committed directly to GitHub, triggering a Netlify deploy

### Manual Editing

Create a markdown file in `content/tools/<category>/`:

```markdown
---
name: "Tool Name"
category: "development"
subcategory: "IDE"
description: "Short description of the tool"
url: "https://example.com"
icon: "https://s3.macosicons.com/..."
tags: ["tag1", "tag2"]
devices: ["mbp", "studio", "ios"]
display: true
status: "active"
featured: false
---

Why I use this tool (optional markdown body).
```

**Categories**: system, development, productivity, communication, media, security, specialized, ai

**Icons**: macOS-style icons from [macosicons.com](https://macosicons.com) CDN. Run `MACOSICONS_API_KEY=<key> npm run fetch-icons` to auto-populate icon URLs.

## Deployment

The site is deployed on **Netlify** with automatic builds on push to `main`.

**Required Netlify environment variables**:
- `TINA_CLIENT_ID` — from Tina Cloud dashboard
- `TINA_TOKEN` — read-only content token from Tina Cloud
- `TINA_SEARCH_TOKEN` — search indexer token from Tina Cloud

## Tech Stack

- **Handlebars** — Templating
- **TinaCMS** — Content management (Tina Cloud)
- **Express** — Development server
- **Chokidar** — File watching
- **Markdown-it** — Markdown processing
- **Netlify** — Hosting and CI/CD
